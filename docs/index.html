AI Glossary AIエージェント 基礎用語

初めてAIエージェントに触れる方へ。よく登場するキーワードをわかりやすく解説します。

🤖
AI Agent
AIエージェント
目標を与えると自分で計画を立て・ツールを使い・複数ステップのタスクを自律的にこなすAIシステム。「指示を受けて実行するだけ」のチャットAIと違い、判断と行動を繰り返します。
🧠
LLM
大規模言語モデル
大量のテキストで学習した、言語を理解・生成するAIモデル。Claude・GPT・Geminiなどが代表例。エージェントの「頭脳」として機能します。
🔧
Tool Use
ツール呼び出し
LLMが外部の機能（検索・コード実行・ファイル操作など）を呼び出す仕組み。これによりAIは「会話するだけ」でなく、実際に作業を行えるようになります。
🔄
RAG
検索拡張生成
Retrieval-Augmented Generation の略。AIが回答を生成する前に外部データベースを検索し、最新情報を取得。LLMの「知識の古さ」問題を補完します。
📋
Context Window
コンテキストウィンドウ
LLMが一度に処理できる情報量の上限（トークン数）。最新モデルでは100万トークン超まで拡大しており、長大な会話や文書も扱えます。
⚡
MCP
モデルコンテキストプロトコル
AIモデルが外部ツールやデータソースに接続するための標準プロトコル（Anthropic 提案）。USB-C のように、AIと各サービスをプラグイン形式でつなげます。
💾
Memory
メモリ（記憶）
エージェントが過去の会話・作業内容・学習結果を保存・参照する仕組み。短期（コンテキスト内）と長期（外部ストレージ）があり、長期メモリでセッションをまたいだ学習が可能になります。
📜
Rules
ルール
エージェントが従うべき行動指針や制約を記述したファイル。「トーン・文体の統一」「禁止ワード」「出力フォーマット」などをあらかじめ定義し、品質と安全性を保ちます。
📦
Skills
スキル
エージェントが再利用できる「手順書」のようなもの。情報収集・記事執筆・公開などの処理をスキルとして定義しておき、必要な場面でエージェントが呼び出して使います。
🪝
Hooks
フック
エージェントの実行ライフサイクルの特定タイミング（開始前・ツール使用後など）に自動で割り込む処理。ログ記録・バリデーション・後処理などをコードで差し込めます。
🤝
Subagents
サブエージェント
オーケストレータ（指揮役）から委譲された特定のタスクを担当するエージェント。「調査担当」「執筆担当」などに専門化することで、複雑なワークフローを並列・分業で処理できます。
👥
Agent Teams
エージェントチーム
複数のエージェントがそれぞれの専門役割を担い、協調して一つの目標を達成する構成。オーケストレータがタスクを振り分け、並列実行や依存関係の管理も行います。
How It Works AIエージェントの動き方

「ゴールを伝えたら、あとは自動でやってくれる」——その裏側で何が起きているのか、4ステップで見てみましょう。

01
指示の受け取り
ユーザーから目標やタスクを受け取り、LLMが内容を解釈。あいまいな指示でも文脈から意図を推定します。
02
計画の立案
目標をサブタスクに分割し実行順序を決定。ReActやChain-of-Thoughtなどの推論手法を活用します。
03
ツールの実行
検索・コード実行・API呼び出し・ファイル操作など、必要なツールを選択して実行。結果をフィードバックとして受け取ります。
04
評価と反復
実行結果をもとにゴール達成を確認。不十分な場合は計画を修正して再実行。目標まで自律的にループします。
