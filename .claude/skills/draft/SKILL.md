# 下書き生成ワークフロー

## 目的

`sources/collected.md` の収集済み情報をもとに、記事の下書きMarkdownを生成して `drafts/` に保存する。

## 手順

### Step 1: ソースを確認する

`sources/collected.md` を読み込み、「記事化候補: Yes」の項目を抽出する。

$ARGUMENTS があればそのテーマに関連するソースを優先する。

### Step 2: 記事構成を決める

以下のどちらかの形式を選ぶ:

**形式A: まとめ記事**（複数ソースを統合）
- 「今週のAIエージェント注目ニュース」などのまとめ形式
- 3〜5件のソースを1記事にまとめる

**形式B: 深掘り記事**（1つのトピックを詳しく解説）
- 特定のツール・手法・事例を詳しく説明する
- 1〜2件のソースをもとに独自解説を加える

### Step 3: 下書きを生成する

CLAUDE.md の記事フォーマットに従って生成する。**形式A・Bで出典ブロック構造が異なる**ので注意。

#### 形式A（まとめ記事）のテンプレート

構成の順番: **タイトル → この記事でわかること（サマリ＋図） → 各出典ブロック**

「この記事でわかること」を冒頭に置くことで、読者が全体像を把握してから個別記事を読める構造にする。

```markdown
---
title: "タイトル（30〜50字）"
date: YYYY-MM-DD
tags: [適切なタグを2〜4個]
summary: "1〜2文の要約"
---

## この記事でわかること

この記事で取り上げるトピックを1行ずつ紹介する。読者が「読むかどうか」を3秒で判断できる情報量にする。

- **[トピック1の短いタイトル]** — ○○という発表があり、□□が変わります
- **[トピック2の短いタイトル]** — ○○が新しくなり、□□に役立ちます
- **[トピック3の短いタイトル]** — …

### 今回の注目ポイント

収集した出典の中でインパクトが最も強いトピックを1つ選び、概念を視覚的に補足する。
記事の種類に応じて以下のいずれかの図を使う：

- **モデル比較・スコア記事** → ASCII 棒グラフや比較表
- **ワークフロー・手順記事** → ASCII フロー図（A → B → C 形式）
- **概念・仕組み記事** → テキストベースの概念マップ（中心語 ← → 関連語）

図の例（モデル比較の場合）:
```
モデル性能比較（Terminal-Bench 2.0）
GPT-5.5        ████████████████████ 82.7%
Claude Opus 4.7 ████████████████░░░░ 69.4%
（前世代平均）  ████████████░░░░░░░░ 55.0%
```

図の例（フロー図の場合）:
```
情報収集 → 要約・整理 → 下書き生成 → レビュー → 公開
  ↑                                        │
  └────────── フィードバックで改善 ─────────┘

#### 形式B（深掘り記事）のテンプレート

```markdown
---
title: "タイトル（30〜50字）"
date: YYYY-MM-DD
tags: [適切なタグを2〜4個]
summary: "1〜2文の要約"
source: "[記事タイトル](URL)（サイト名）"
---

## はじめに

（200字程度：背景と記事で扱う内容を明示）

## [H2セクション1]

（本文 300〜600字）

> **初心者向けまとめ**: このセクションで覚えておくべきことを1〜2文で平易にまとめる。

## [H2セクション2]

…

## まとめ

（200字程度：要点の整理と次のアクション）
```

### Step 4: drafts/ に保存する

ファイル名: `YYYY-MM-DD-[slug].md`
例: `2026-04-14-agent-news-weekly.md`

### Step 5: 完了報告

保存先パス・記事タイトル・文字数・使用したソース一覧を出力する。
次のステップとして `/review` の実行を案内する。
